# GridLock UI Revamp Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Transform the GridLock F1 2026 Dashboard from a basic design-token showcase into a cinematic, F1 broadcast-inspired experience with 3D elements, GSAP scroll animations, and a futuristic glassmorphism UI.

**Architecture:** GSAP-first animation approach with lightweight 3D (React Three Fiber) for the hero car only. Server components handle data fetching; thin client wrappers add GSAP animations. All new 3D/animation code is lazy-loaded and code-split.

**Tech Stack:** Next.js 16, React 19, GSAP 3.14 + ScrollTrigger, Three.js + React Three Fiber + Drei, Framer Motion, Tailwind CSS v4, shadcn/ui

**Files:**

- [ ] **Step 1: Create GSAP utility module**

Create `src/lib/gsap.ts` with:
- Register `ScrollTrigger` plugin via `gsap.registerPlugin(ScrollTrigger)`
- Export `animateCounter(element, target, duration?)` ŌĆö tweens `textContent` from 0 to target
- Export `staggerEntrance(selector, container)` ŌĆö fades in + slides up children with 0.08s stagger, triggered by ScrollTrigger
- Export `barFill(element, targetWidthPercent)` ŌĆö tweens width from 0 to target%, ScrollTrigger-activated
- Export `numberFlip(element, from, to)` ŌĆö digit flip via y-offset animation

```typescript
"use client";

import gsap from "gsap";
import { ScrollTrigger } from "gsap/ScrollTrigger";

gsap.registerPlugin(ScrollTrigger);

export { gsap, ScrollTrigger };

export function animateCounter(
  element: HTMLElement,
  target: number,
  duration = 1.5
) {
  return gsap.to(element, {
    textContent: target,
    duration,
    ease: "power2.out",
    snap: { textContent: 1 },
    scrollTrigger: {
      trigger: element,
      start: "top 85%",
    },
  });
}

export function staggerEntrance(selector: string, container: HTMLElement) {
  return gsap.from(selector, {
    y: 40,
    opacity: 0,
    duration: 0.6,
    stagger: 0.08,
    ease: "power3.out",
    scrollTrigger: {
      trigger: container,
      start: "top 80%",
    },
  });
}

export function barFill(element: HTMLElement, targetWidthPercent: number) {
  return gsap.fromTo(
    element,
    { width: "0%" },
    {
      width: `${targetWidthPercent}%`,
      duration: 1.2,
      ease: "power2.out",
      scrollTrigger: {
        trigger: element,
        start: "top 85%",
      },
    }
  );
}
```

## Key Technical Decisions

| Decision | Rationale |
|----------|-----------|
| Use `@gsap/react` useGSAP directly | Official hook handles cleanup, SSR, context. No wrapper needed. |
| Drei primitives instead of GLTF | Zero network requests, easy to tweak, no licensing issues |
| GSAP drives Three.js car position | Master timeline coordinates 3D + DOM elements in sync |
| Server components + thin client wrappers | Keep data fetching on server, add interactivity via small client components |
| `frameloop="demand"` after hero | Stop R3F render loop when car is static (performance) |
| CSS transitions for card hovers | More performant than JS-animated hovers for many cards |
